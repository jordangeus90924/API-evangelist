The API Specification Toolbox

The API Specification Toolbox

There are a handful of specifications available today that help API providers better describe the surface area of their APIs, providing a common vocabulary that can be used to design and deliver an API, while also helping evolve and manage those APIs across the entire API lifecycle. This is a walk through of each of the specifications and how they are being applied across a single API, helping demonstrate how they work together.

OpenAPI

This is the OpenAPI for the Slack web API, definiing the surface area of the web API, making each of the available API paths available in a machine readable format, leveraging a common specification to define a single instance of a request and response API.
